Battlecode 2015
Programming Competition | 2015
Battlecode is a programming competition held over IAP (Independent Activities Period)-which is the entire month of January. A game in Battlecode pits two competitor programmed artificial intelligences against each other in a game of real time strategy. In 2015, The object of the game was to destroy the enemy headquarters. Each team has up to six towers on the map, and destroying these towers weakens the headquarters. 2015 was a year of incredibly imbalanced units. We had 6 different fighting units and it was horribly unfair in the sprint tournament where only one unit was viable. Then in the final tournament only one unit was any good again, though it depended on the map whether you could just open into it.
Supply
Supply necessitated that you had a good system for distributing it. We decided to make a double queue system out of the radio system (a global static int array) where army had priority over miners because army was what kept you from dying in the early game.
Mining
Miners mined at pretty much the same rate as long as the mineral patch they were on was above a certain threshold. For this reason we had miners search the closest mineral patches first because traveling time ended up being a huge factor.
Build order
Because this game was very similar to starcraft we created a build order class so that we could easily specify an exact build order. This was important because the top teams worked out the exact best builds with precision down to the turn.
